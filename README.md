# Liquid-Cat

Liquid-Cat is a project workspace for preserving and applying the Liquid Cat method: a structured exploration mode that lets ideas stretch into unusual shapes without losing their underlying structure.

## Purpose

Liquid-Cat exists to make creative jumps safer, clearer, and more reusable.

It is designed for moments when two ideas look unrelated at first, but may share a hidden structural path. The method explores those paths without forcing premature categories, then keeps only the routes that remain coherent under review.

## Core principles

- Explore before drawing hard boundaries.
- Keep the original structure intact while testing unusual combinations.
- Prefer reusable paths over one-off cleverness.
- Treat local fixes as local fixes, not global completion.
- Separate observation, hypothesis, and conclusion.

## Operating modes

### Liquid Cat mode

Use this when exploring a non-obvious connection, metaphor, product idea, prompt design, or conceptual bridge.

Expected output:

1. Observations
2. Candidate paths
3. Structural fit check
4. Useful result or rejected path
5. Remaining open branches

### Pawprint Audit

Use this when checking whether a proposed change is clean, reusable, and non-duplicative.

Audit questions:

- Is there already a valid path that can be reused?
- Did this change preserve the project structure?
- Is this a local patch being mistaken for a complete solution?
- Are there duplicated concepts, files, or instructions?
- Is the next action clear?
